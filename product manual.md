# Cream by Solstar 商品ページマニュアル

商品詳細ページを最短で整えるための実践ガイドです。ホームページ版と同じ構成で、商品テンプレートに特有のセクションだけを抜粋しています。

## まずはここから（基本操作）
- テンプレート切替: エディタ上部のプルダウンで「商品」を選択し、任意の商品をプレビュー対象に指定。
- 商品の切替: 右上の「商品を変更」で想定バリエーション数や在庫状態が近い商品を選び確認。
- ブロック編集: 「商品情報」セクション内のブロック構造を入れ替えて、見出し→価格→バッジ→フォームの順番を調整。
- 下部セクション: 商品ページでも一般セクションを追加できますが、本マニュアルでは商品専用テンプレートに絞っています。
- 反映確認: PC/モバイル両方でビューを切替え、スティッキーATCやギャラリー比率を実機幅で確認。

## セクション共通の見た目調整
- ここを触る: 各セクション右側の「セクション設定」タブ、またはブロックごとの設定パネル。
- よく使う項目: セクション幅、カラースキーム、上下余白、列間隔、ギャラリー比率、ブロック内パディング。
- ヒント: 商品ページは余白を詰めすぎると縦長になるため、デスクトップは16-24pxのギャップ、モバイルは12px目安で揃えると読みやすくなります。

## テンプレート
- 本ガイドで紹介するのは「商品詳細ページでのみ利用できるセクション」です。共通セクション（テキスト、メディア付きテキスト等）はホームページマニュアルを参照してください。

### 商品情報
- 目的: メディアギャラリーと購入エリアを並べ、商品説明からCTAまでを1セクションで完結させる。
- 含まれる機能: メディアギャラリー（グリッド/スライド）、タイトル・価格グループ、バッジ、タブ/アコーディオン説明、サイズチャートポップアップ、バリエーション&スウォッチ、購入ボタン、アプリブロック。
- 使う場面: PDPのメインコンテンツとして常設。ギャラリー左右入替や詳細エリアのスティッキー化でレイアウトを調整。

設定詳細
| 設定 | 種別 | 説明 |
| --- | --- | --- |
| セクション幅 | Select | ページ幅/全幅を選択。全幅にすると等分レイアウト時に左右いっぱいまで広がる。 |
| メディア位置 | Select | PCでメディアを左/右に配置。右を選ぶと商品の詳細を先に見せたい場合に有効。 |
| カラムバランス | Checkbox | 「列を同じ幅にする」「詳細幅を制限」でギャラリーと詳細を1:1やサイドバー幅に固定。 |
| ギャップ | Range(px) | ギャラリーと詳細間の余白。16-32pxで視線誘導がしやすい。 |
| カラースキーム | Color scheme | 背景と文字色のセット。ヒーローと同一配色にすると世界観が揃う。 |
| 上下余白 | Range(px) | セクション外周の余白。ヘッダー直下は上余白0にして詰めるケースが多い。 |

ブロックのポイント
- メディアギャラリー: グリッド/スライド、列数、サムネイル位置、アスペクト比、ズーム、動画ループ、サムネ幅、ビューポート内での拡張を選択可能。
- 商品詳細ブロック: 詳細列の幅やスティッキーON/OFF、枠線/背景メディア/角丸を調整。背景メディアを設定するとカード型の購入ボックスが作れる。
- Variant picker: ボタン/ドロップダウン切替、スウォッチ有無、整列（左/中央）を設定。サイズが多い場合はボタンよりドロップダウン推奨。
- Popup link: サイズチャートや配送ポリシーなどの静的ページをモーダルで開ける。リンク文言を商品カテゴリごとに変えておくと親切。
- Tabs & Accordion: 詳細説明をタブ/折りたたみで整理。タブは最大5項目、アコーディオンはアイコンと開閉状態を設定可。長文は「続きを読む」を併用して折り返し。
- Buy buttons: 数量欄、ギフトカード入力、Shop Payボタンなどを並べ替え可能。pickup availabilityをONにすると店舗受取情報が表示される。

Tips
- ギャラリーをグリッド表示にする場合は「第一画像を拡大」をONにすると1枚目だけ大きく見せられる。
- バリエーション画像が多い商品は「バリエーション画像を隠す」をOFFにしておくと切替時に別画像が表示される。
- Sticky detailsはPCのみ有効。ヒーローの高さと重ならないようヘッダー高さ分のオフセットが自動計算される。

### スティッキーATC（sticky-add-to-cart）
- 目的: スクロールで購入フォームが画面外に出たあとも、画面下部からバリエーション選択と購入ボタンを常時表示。
- 含まれる機能: 商品サムネイル/タイトル/価格、ドロップダウン式バリエーション選択、数量調整、Add to Cartボタン、モバイル専用モード。
- 使う場面: コンテンツ量が多いPDPやモバイルでのスクロール距離が長い場合にCVRを維持。

設定詳細
| 設定 | 種別 | 説明 |
| --- | --- | --- |
| セクション有効化 | Checkbox | 商品ページかつ在庫あり時のみ表示。構築中は無効化してスクロール挙動を個別確認。 |
| モバイルのみ | Checkbox | デスクトップを除外する場合に使用。 |
| カラースキーム | Color scheme | バー全体の背景/文字色。透過はカスタムCSSで制御。 |
| バリエーション/数量/画像表示 | Checkbox | それぞれの表示可否を切替。バリエーションOFF時は現在選択されているSKUをそのまま使用。 |
| 上下パディング | Range(rem) | バー内部の縦余白。モバイルで0.5rem、デスクトップで1rem程度が推奨。 |
| 注意書き | Paragraph | フッター固定メニュー使用時のクリアランスに関するリマインダー。 |

Tips
- Bottom Menuと併用するときは、セクションの注意書きどおりバーの位置が自動で調整されるかを実機で検証。
- バリエーション項目が多い場合、スティッキーATCでは自動でドロップダウン化されるため命名を短く整えておくとUIがすっきりする。

## 連携・注意事項
- Product RecommendationsはSearch & Discoveryアプリの設定を参照します。Complementaryタイプを使う場合はアプリ側でペアリングを作っておく必要があります。
- Popup LinkやTabsに外部アプリブロックを挿入する場合、表示崩れが起きやすいので1ブロックずつ追加して確認しましょう。

## Tips（運用のコツ）
- 画像: メインギャラリーは縦長(3:4)×4枚をベースにし、動画は最後尾に置くとカルーセル操作がシンプル。
- コピーワーク: タブの見出しは名詞+一語（例: 素材、ケア）で揃えると国際対応しやすい。
- 変数確認: 価格表示が税込/税別で揺れないよう、priceブロックの「税込表記」設定を本番前に再チェック。
- パフォーマンス: メディアを16MB以下に抑え、ズームON時は2048px以上の画像をアップロード。

## 公開前チェックリスト
- バリエーション切替: 画像・価格・在庫・SKU表示が全組み合わせで正しく更新されるか。
- サイズチャート: Popup Linkが正しいページを参照しているか、翻訳済みか。
- タブ＆アコーディオン: 表示/非表示の文字数がバラつかないか、折返し高さを調整したか。
- スティッキーATC: 在庫切れ状態やログアウト状態でも期待通りの表示になるか。
- レコメンド: Search & Discovery設定と一致するか、在庫切れ商品が混ざっていないか。
