基本操作

      • 管理画面: 「オンラインストア > テーマ > カスタマイズ」から編集します。
      • テンプレート切替: 上部プルダウンでホーム/商品/コレクション/ブログ/検索/カートなどを選択。
      • セクション編集: 左サイドバーでセクション/ブロックを追加・並べ替え・非表示・削除。
      • メディア管理: 画像/動画は「設定 > ファイル」にアップロードして選択。
      • 反映確認: 保存後にプレビュー/公開で確認。

      Header

      • サイト最上部で告知とナビゲーションを担う領域。ブランドの第一印象と主要導線をここで整えます。

      • 表示: 画面上部でメッセージをスライド表示。
      • 主な設定: 切替間隔、幅、配色、区切り線。
      • 使う場面: セール告知や配送案内を短文でループ表示したいとき。

      テンプレート

      • 下記はHomepageで利用できるテンプレート（セクション）に限定したガイドです。コレクションページやブログページ専用のテンプレート機能は本マニュアルでは非表示にしています。

      コレクション

      • 目的: ホームで推すカテゴリやコレクション導線を並べ、回遊を促す。
      • 含まれるセクション: 特集コレクション、コレクションリスト、コレクションリンク、タブコレクション。

      特集コレクション

      • 表示: 指定コレクションの商品一覧をカードで訴求。
      • 使う場面: トップで推したいカテゴリの紹介に。

      設定詳細

      設定 種別 説明
      コレクション Collection 表示するコレクション。
      レイアウト Select(grid/carousel) グリッドか横スライドか。
      列数/モバイル列 Range PCとモバイルの列数。
      商品数 Range 最大表示件数。
      タイトル行数制限 Range 商品名の折返し行数。

      コレクションリスト

      • 表示: 複数コレクションをグリッド/カルーセルで一覧化。
      • 使う場面: ギフト/価格帯/テーマ別など複数カテゴリを早見で並べたいとき。
      • レイアウト種別: グリッド/カルーセル/Bento/エディトリアル。

      基本操作

      • 追加場所: テーマエディタで任意のテンプレートに「コレクションリスト」を追加。Heroやコピーの直下に置くと効果的。
      • 対象コレクション: 表示したいコレクションを複数選択。未選択の場合はレイアウト種別ごとの上限数でプレースホルダーが生成。
      • レイアウト切替: grid / carousel / bento / editorial を切り替えながらPC/モバイルプレビューを確認。
      • リンク先確認: 各カードは該当コレクションURLへ遷移。URL変更はコレクション側で一括反映。

      共通デザイン設定

      • ここを触る: 右サイドバー「セクション設定」。カード数・レイアウト・ギャップ・セクション幅・カラースキームをまとめて調整。
      • よく使う項目: layout_type、columns、carousel_on_mobile、section_width、gap、上下余白。
      • ヒント: section_widthをpage-widthにすると他セクションと左右余白が揃いやすい。全幅の場合はブロック側で余白を追加すると窮屈さを防げる。

      設定詳細（共通）

      設定 種別 説明
      コレクションリスト Collection list 表示したいコレクションを複数選択。空の場合はレイアウト種別別の最大数までプレースホルダーが生成。
      レイアウトタイプ Select(grid/carousel/bento/editorial) grid / carousel / bento / editorial。選択内容に応じて後続オプションが変化。
      セクション幅 Select page-width / full-width。Hero直下はpage、連続するコレクション群はfullが目安。
      セクションギャップ Range(px) セクション内上下の要素間余白（ヘッダーとカード群の距離）。
      列数/モバイル列/間隔 Range グリッドの設定。
      カード背景/アイコン Color/Image カードの飾りを設定。
      表示数 Range 最大表示件数。
      カラースキーム Color scheme 背景と文字色セット。ブロック内のテキスト/ボタンも継承。
      上下余白 Range(px) セクション外周の余白。前後に同系色セクションがある場合は上下16px程度に詰めると一体感が出る。

      各レイアウトの概要

      • Grid: columns（1-8列）、mobile_columns（1or2列）、columns_gap（横）、rows_gap（縦）、carousel_on_mobile（モバイルだけスライダー化）。
      • Carousel: columnsはスライド内カード数。mobile_card_sizeで60cqw/44cqwを選択。icons_styleとicons_shapeで矢印/シェブロン＋円/角/なしを設定。
      • Bento: bento_gapでタイル間余白を管理。最大4枚のコレクションを不均等グリッドで配置。
      • Editorial: max_collections（1-16）を手動指定し、余白広めのカードを段組表示。文章ブロックでストーリー性を出す用途に向く。

      ブロックの使い方

      • 追加できるブロック: テキスト/画像/アイコン/ボタン/グループ/動画/区切り等。セクション冒頭にヘッダーを作ると管理しやすい。
      • カスタムCTA: ボタンブロックで「すべての商品を見る」リンクを設置。section_widthがpage-widthならブロック幅もfillで左右整列。

      Tips

      • カルーセルのcolumnsをPC4/モバイル1にし、carousel_on_mobileをOFFにするとPCはスライダー、モバイルは横スクロールカードで視認性が高い。
      • Bentoレイアウトは最大4枚。配置するコレクション画像は縦横比を揃えると連続性が出る。
      • Editorialはコレクション未選択でもmax_collections分のプレースホルダーで骨組み確認が可能。
      • 画像の代わりにカラー背景だけでカードを作りたい場合は、代表画像を単色に差し替えるか画像内にテキストを入れてタイル風に。

      連携・注意事項

      • 表示順は「コレクションリスト」で選択した順番に従い、管理画面のコレクションソートは影響しない。
      • コレクションに画像が未設定の場合はプレースホルダーが表示されるため、公開前に代表画像を登録。
      • カルーセルのナビアイコンはテーマ全体のIconフォントを利用。独自SVGはsnippets/resource-list-carousel.liquid側のカスタマイズが必要。

      公開前チェックリスト

      • コレクション選択: 非公開コレクションが混ざっていないか、画像が高解像度で揃っているか。
      • レイアウト確認: Grid/Carousel/Bento/Editorialの切替で余白や行数が崩れないか。
      • モバイル: 矢印やドットが押しやすいか、プレビューで確認。
      • ブロック: 見出し・説明・ボタンのフォント設定やリンク先を再確認。
      • 翻訳/アクセシビリティ: ラベルの翻訳表示と、コレクション画像の代替テキストを確認。

      Tips（運用のコツ）

      • 画像/動画: ヒーローは横長16:9か4:5、高さに迷ったら70vh。WebP優先、動画は短尺mp4(H.264)、モバイルは軽量版を用意。
      • 配色とロゴ: カラースキームごとにロゴ（通常/逆さ）を確認。暗い背景では逆さロゴを使う。
      • モバイル最適化: 列数を減らし余白を詰める。追従メニューを有効化し、ヒーローはモバイル画像を別設定。
      • パフォーマンス: 不要なポップアップやオートプレイをオフ。画像は事前リサイズ、動画は音声オフ＋短尺を前提に。
      • アクセシビリティ: 画像代替テキストを入れる。ボタン文言は動詞から。コントラストが低い配色は避ける。
      • 多言語/多通貨: ローカライズスイッチを有効にし、言語別ロゴ/コピーを確認。静的テキストは言語別で複製して出し分ける。
      • 商品訴求: バリエーションが多い商品はスウォッチサイズを調整。追従カート＋クイック追加を併用するとCVR向上が期待できる。
      • 検索/フィルター: Search & Discoveryでフィルター/並び替えを設定し、空状態メッセージを編集して回遊先を提示。
      • よくあるつまずき: 地図はAPIキー未設定だと非表示。ニュースレターポップアップはPC/モバイル両方でテスト。動画が重いと再生が遅くなる。

      公開前チェックリスト

      • ナビ/フッター: リンク切れなし、法務リンクとコピーライト更新、ロゴ視認性を各配色で確認。
      • 配色: カラースキーム割当と文字コントラストをチェック。
      • モバイル: 列数・余白・フォント、追従メニュー、ヒーローのモバイル画像を確認。
      • 商品ページ: バリエーション選択、スウォッチ、追従カート、クイック追加、価格/通貨表示を確認。
      • 検索/フィルター: Search & Discoveryの設定と空状態メッセージを確認。
      • カート/チェックアウト導線: メモ・割引・送料表示、ボタン文言、挙動を確認。
      • システムページ: 404とパスワードの文言と導線を差し替え済みか。
      • パフォーマンス: 不要セクション/ポップアップを無効化、画像圧縮、動画は短尺に。
