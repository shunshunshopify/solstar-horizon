# Cream by Solstar ブログテンプレートマニュアル

ブログ一覧（/blogs/…）と記事詳細（/blogs/.../articles/...）テンプレート専用のガイドです。カードレイアウトや記事本文の幅、コメント設定など、ブログ運用に必要な項目をまとめています。

## まずはここから（基本操作）
- テンプレート切替: テーマエディタ上部から「ブログ」「ブログ記事」を選択し、対象のブログ/記事をプレビューに指定。
- テストデータ: ブログ一覧は最低3本以上の記事があるとレイアウトの変化（1列→2列→3列）が確認しやすい。ダミー記事を用意して動作をチェック。
- 画像確認: 記事にアイキャッチを登録しない場合はカード外枠が表示されるため、全記事に横長画像を登録しておくのが推奨。
- コメント/タグ: Shopify管理画面でコメント許可やタグ付与を設定してからテンプレートを編集すると、UIとの連携が把握しやすい。

## セクション共通の見た目調整
- ここを触る: `main-blog`・`main-blog-post`のセクション設定。
- 項目: カラースキーム、上下余白、ギャップ、セクション幅。
- ヒント: 記事詳細はテキスト中心のため、`section_width = page-width`＋ギャップ32pxで読みやすくなります。背景を変えたい場合はカラースキームをブログ専用に用意してください。

## テンプレート

### ブログ一覧（main-blog）
- 目的: 指定ブログの最新記事をグリッドで表示。`blog-posts-list.js`が自動でグリッドを組み、12件ずつページネーションを実行。
- 構成: タイトルブロック＋`_blog-post-card`（固定レイアウト）。アプリ/テキストなど追加ブロックを冒頭に差し込み可能。

設定詳細
| 設定 | 種別 | 説明 |
| --- | --- | --- |
| カラースキーム | Color scheme | 背景・文字色セット。タイル枠線やカード内テキスト色にも反映。 |
| 上/下余白 | Range(px) | ページ全体の余白。ヒーロー直下で使う場合は上余白を0-16pxに調整。 |
| タイトルブロック | Text | デフォルトで`<h1>{{ blog.title }}`。フォント/位置/余白を個別調整してセクション導入に使用。 |
| 記事カード | Static | `_blog-post-card`内の設定で、タイトルプリセット、日付/著者表示、説明文フォント、画像の高さ（small/medium/large）などをカスタマイズ。 |

動作メモ
- グリッド1行目は1件、2行目は2件、以降は3列レイアウトに自動変化。記事数が少ない場合でもカードサイズは比率に合わせて拡大します。
- ページネーションは標準の`paginate`で12件ずつ。読み込み位置は`viewMorePrevious/Next`スパンで制御されているため、Lazy loadを追加する場合は`blog-posts-list.js`側の改修が必要です。
- カードに画像が無い場合は自動でボーダー＋余白つきのプレーンカードになります。

### ブログ記事（main-blog-post）
- 目的: 記事タイトル・メタ情報・アイキャッチ・本文・コメントフォームを1つのセクションで表示。
- ブロック構成: タイトル（text）、詳細（_blog-post-info-text）、アイキャッチ（_blog-post-featured-image）、本文（_blog-post-content）。

設定詳細
| 設定 | 種別 | 説明 |
| --- | --- | --- |
| ギャップ | Range(px) | タイトル、画像、本文、コメントの縦間隔。32px前後が読みやすい。 |
| カラースキーム | Color scheme | 記事背景とテキスト色。本文は`page.content`同様にCSSクラスを継承。 |
| 上下余白 | Range(px) | セクション全体の余白。Hero型記事では上余白0、下80pxなどで余裕を持たせる。 |
| タイトルブロック | 固定 | Liquidで`article.title`を出力。フォント/揃えをカスタム可。 |
| 詳細ブロック | `_blog-post-info-text` | 投稿日/著者の表示可否と整列を設定。タグ表示を追加したい場合はスニペットを拡張。 |
| アイキャッチ | `_blog-post-featured-image` | 比率（landscape/squareなど）と枠線/角丸/パディングを設定。 |
| 本文 | `_blog-post-content` | `width_mode = fixed`で読みやすい幅に制限。fluidに切替えると画面幅いっぱいに広がる。 |

コメント
- ブログでコメント有効化時、記事末尾に一覧とフォームが表示。10件ずつページネーションされます。
- フォーム文言やバリデーションは翻訳キー`blogs.article.comment_*`に準拠。承認制の場合は送信後のメッセージを忘れずに翻訳。

アプリブロック
- main-blog-postの末尾 `content_for 'blocks'` には@appブロックのみ挿入可能。レビューウィジェットやレコメンドを追加したい場合に利用。

## 連携・注意事項
- 記事本文・抜粋はShopify管理画面で編集。テーマからHTMLを直接編集する場合は`_blog-post-content`スニペットを拡張する。
- カテゴリ導線を追加したい場合は、ブログテンプレートで`text`ブロックを追加し、タグ/カテゴリーリンクを手動で設置。
- RSSや構造化データは`article | structured_data`で出力済み。追加のJSON-LDが必要な場合は記事セクション内で拡張してください。
- パンくずは両テンプレートで自動レンダリング。非表示にする場合は`snippets/breadcrumbs.liquid`側で制御。

## Tips（運用のコツ）
- 画像管理: アイキャッチは横長(3:2〜16:9)で揃えるとカードレイアウトが整います。WebP優先、幅1200px以上推奨。
- タイトル長: 一覧カードの`type_preset`を小さめに設定し、タイトルを40文字以内に収めると3列レイアウトでも折返しが美しくなります。
- 目次/アンカー: 記事本文で長文になる場合はリッチテキストにリンクリストを作成し、`id`属性をヘッダーに追加すると読者がスクロールしやすい。
- CTA: 記事末尾にアプリブロックを使い、ニュースレター登録や関連商品バナーを表示して回遊を継続。

## 公開前チェックリスト
- 一覧ページ: 画像欠けやタイトルの折返しが不自然でないか、モバイルで列が壊れていないか。
- ページネーション: 12件以上の記事でページ送りが表示され、リンク先URLが正しいか。
- 記事ページ: アイキャッチ、本文、引用、埋め込み（動画/iframe）がレイアウトを崩していないか。
- コメント: 有効化時にフォーム送信→承認フロー→公開まで問題ないか。スパム対策（reCAPTCHA）が有効か。
- メタ情報: 著者名/日付の表示ルールがブランドトーンに合っているか、翻訳が済んでいるか。
